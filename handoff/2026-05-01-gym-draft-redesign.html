GYM DRAFT — REDESIGN MOCKUP

2026-05-01 · Focus: final 2 picks (nominating phase) · Pair this file with handoff/2026-05-01-gym-draft-audit.md
What's new (over the current implementation):
  • Vote LED row — 1 cell per player, live yes / no / pending state. Replaces the silent "do you agree?" buttons.
  • Live tally + threshold — "2 YES · 1 NO · 1 PENDING — needs 3/4 to pass."
  • Tiebreak story — 2-2 ties give the deciding vote to the first-pick captain (★). Real rule, real UI.
  • Rejection rotation — rejected nomination auto-advances to the next nominator with a visible breakdown (current code reuses the same nominator forever).
  • Skip-turn restricted to current nominator — current code lets anyone skip anyone's turn (security/UX bug).
  • Change-vote allowed pre-resolution — current JS disables vote buttons after click; the model permits a change but UI doesn't.
