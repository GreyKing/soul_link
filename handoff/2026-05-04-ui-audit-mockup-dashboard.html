R1 — Dashboard Shell + Tab Navigation

Main dashboard `/` · 2026-05-04 audit · architect-only redesign pass
What changes here
  • Tab bar becomes a real tablist — `role="tablist"` + `role="tab"` + `aria-selected` + `aria-controls`. Icon + text + active underline + (optional) live-update dot.
  • Title bar slimmed: stats become a single inline strip, run dropdown becomes a pill with chevron showing the current run number + status pill (ACTIVE / VIEWING / COMPLETE). One row instead of two.
  • Right rail consolidates today's three stacked panels (Party / Map / Gyms) into one card with sub-tabs. Saves ~200 vertical pixels and removes the duplication between sidebar and main-tab content.
  • Mobile: layout collapses to single column — title bar → tab bar (horizontal-scrollable) → main panel → right rail at the bottom. Left party sidebar folds into the main PARTY tab on phones.
  • Tablet: two columns — left party sidebar merges into PARTY tab; main panel + right rail side-by-side.
  • Keyboard navigation: ← / → moves between tabs; 1–7 jumps directly. Esc closes any open modal.

SCREEN 6

Tab focus + ARIA spec sheet

For implementers — exact ARIA wiring for the tablist + tabpanels. Today's tabs use class-based active state with no semantics; this brings them up to standard.

TAB BAR
<div role="tablist" aria-label="Dashboard sections">
  <button role="tab" id="tab-pcbox"
          aria-selected="true"
          aria-controls="panel-pcbox"
          tabindex="0">
    <span class="icon">▦</span> PC BOX
  </button>
  <button role="tab" id="tab-map"
          aria-selected="false"
          aria-controls="panel-map"
          tabindex="-1">
    <span class="icon">⛰</span> MAP
  </button>
  ...
</div>

<section role="tabpanel" id="panel-pcbox"
         aria-labelledby="tab-pcbox"
         tabindex="0">
  ... content ...
</section>
KEYBOARD CONTRACT
  • ← / → — move focus, wrap at edges; aria-selected updates immediately
  • Home / End — first / last tab
  • 1–7 — jump (only when no input has focus)
  • Tab — moves into the active panel (skipping inactive tabs because of tabindex="-1")
  • Esc — close any open modal; falls through to cancelling drags if no modal
FOCUS MGMT

Activating a tab moves focus to the tab itself (not the panel — WAI-ARIA recommends user opts in via Tab). The panel has tabindex="0" so it's reachable but not auto-focused.

STIMULUS WIRING

Single Stimulus controller `tablist`, two targets: tab + panel. Action click->tablist#activate keydown->tablist#navigate. URL hash (#pcbox, #map) preserves tab across reloads — already partially supported in `pixeldex_controller.js`; this generalizes it.

Total estimated work: 2–3 days; mostly view/CSS, no model or migration changes. The PARTY and GYMS rendering stays in the existing partials — we're just changing how they're hosted.
