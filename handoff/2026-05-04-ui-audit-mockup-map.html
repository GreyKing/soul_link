R4 — Map / Route Timeline

Standalone /map · 2026-05-04 audit · architect-only redesign pass
What changes here
  • Right-edge gradient fade on the timeline — visual cue that there's more to scroll. Plus a sticky "JUMP TO NOW" pill that anchors scroll on the next-uncaught route.
  • Pulse ring on the next-uncaught node + a "↓ NOW ↓" pin floating above it. The user always knows where to look.
  • Larger touch targets — 84px node minimum (vs. today's 56px). Two-line labels instead of `text-overflow: ellipsis`.
  • Always-visible legend — caught · dead · uncaught · special · gym — top of the page so the iconography is self-explanatory.
  • Slide-out becomes a right-rail sheet (sticky on desktop, half-height bottom-sheet on mobile). Per-player species inputs use the existing combobox controller, not `<datalist>`.
  • Mobile: a per-segment accordion replaces the horizontal track. Open the current segment, see vertical rows. Honest mobile UX.

A · ALWAYS-VISIBLE LEGEND Sat right above the track. Five glyphs, five labels. New users learn the iconography in 3 seconds; returning users glance over it. No tooltip-only knowledge.
B · PULSE RING + "NOW" PIN Two redundant signals on the next-uncaught route. Color (amber pulse), motion (animation), and a literal "↓ NOW" caption. Color-blind safe because the caption + pulse pair is enough on its own.
C · SEGMENT DIVIDERS Vertical line + label between segments. Today's timeline has gym milestones serving as dividers; this preserves them but adds a textual marker for users who want to navigate by major arc, not by gym number.
D · EDGE GRADIENT FADE Two pseudo-elements on the frame: 64px right gradient + 36px left gradient. Visually announces "more here." Plus a translucent scroll cursor visible on hover.
E · "JUMP TO NOW" PILL On the status bar. Subtle pulse animation. One click scrolls the timeline so the pulse-ring node is centered. Reverses by hitting it again (back to leftmost).

A · STICKY SHEET, NOT MODAL The sheet is sticky-positioned to the right column, not an overlay. Users see the timeline and the form simultaneously — common interaction is "look at the route, then fill in catches" without context-switching.
B · PER-PLAYER ROW WITH PREVIEW The 28px sprite preview lights up as soon as the species is recognized. Gives feedback without a "Confirm species" step. Matches today's `speciesPreview` target — controller already exists.
C · COMBOBOX, NOT DATALIST Drop the `<datalist>` (Safari renders it as an OS dropdown that vanishes on tap). Reuse `timeline_controller.js#filterSpecies` which already does a custom dropdown. Consistent UX across browsers.

A · ACCORDION OVER SCROLL The horizontal track makes sense on a 1280px monitor; on a 360px phone it becomes 5 minutes of swiping. Per-segment accordion lets the user collapse what they're done with and focus on the current arc.
B · OPEN-BY-DEFAULT The current segment opens automatically. Past segments collapse. The user lands directly on the row they need.
C · SEG SUMMARY Each summary shows progress (`5/6 ✓`, `1/6 NOW`). Glanceable: progress without expanding. The active segment carries an amber background to draw the eye.
D · TAPPING A ROW On mobile, tapping a row opens the bottom-sheet (full-screen below 480px) instead of a side rail. Same content as the desktop sheet. Submit closes the sheet, refreshes the row in place.
CLOSING NOTE The desktop timeline keeps the same Stimulus controller (`timeline_controller.js`) — view changes are mostly markup + CSS. The accordion fallback is a single ERB conditional + CSS rule (`@media (max-width: 720px) { .timeline-frame { display: none; } .accordion-frame { display: block; } }`). The combobox per player is already implemented — `data-action="input->timeline#filterSpecies"`. So the implementation is mostly visual and IA, not new infrastructure.
