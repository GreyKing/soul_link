R2 — PC Box / Catches

Dashboard PC BOX tab · 2026-05-04 audit · architect-only redesign pass
What changes here
  • Auto-detected catches become a REVIEW TRAY at the top with a one-line badge legend and a per-row primary action (LOG / SKIP / EDIT). The flow ends at a click, not at staring.
  • On-team / storage / fallen unify into one filterable grid with chips. Filtering is client-side; URL hash preserves chip state across reloads.
  • Cells get real hover states (border-color shift + 2px lift), `cursor: pointer`, and a group marker in the corner indicating ON TEAM / STORAGE / FALLEN at a glance.
  • Type-coverage moves to a side rail instead of being a footer. Covered types are filled green; gaps are dashed crimson; shared weaknesses are filled crimson.
  • Mobile reflows to 3-column grid with sticky filter chips at the top of the panel.
  • Search field replaces today's "+ NEW CATCH" with both: NEW CATCH stays as the primary action, search is global.

A · REVIEW TRAY Three things change vs. today: (1) badges are color-coded by intent — green for "log this," amber for "warning," neutral for "informational." (2) Each row has its own primary action — a button, not a label. (3) The recommended action is highlighted; user just hits Enter to accept the default.
B · ONE GRID, FOUR FILTERS No more three separate sections that all use the same cell layout. Filter chips at the top scope the grid. Counts on the chips remove ambiguity. ALL is the default but URL hash (#fallen, #team) preserves filter state for deep links.
C · GROUP MARKER ICON The corner glyph (★ team · ▣ storage · ☠ fallen) makes status legible at-a-glance even when filtered to ALL. Color-blind safe because each shape is distinct. Same icon set the timeline uses — visual vocabulary stays unified.
D · TYPE COVERAGE RAIL Side rail (280px) instead of a footer. Covered/gap/weak read top-to-bottom in priority order: what we have → what we're missing → what kills us. Numbers right on the badges (FIG ×3) so the user doesn't have to count.
