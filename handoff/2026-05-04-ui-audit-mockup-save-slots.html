R3 — Save Slot Management

Emulator left sidebar · 2026-05-04 audit · architect-only redesign pass
What changes here
  • Every slot has a visible STATE PILL (EMPTY · SAVED · ACTIVE · TARGET · CONFIRM). No more hidden overlay for overwrite-pending mode.
  • Overwrite-pending mode is announced by a sticky banner at the top of the column. Targetable slots glow amber. The user explicitly opts in or cancels.
  • DELETE swaps the slot into an inline "Confirm delete?" state — the second click is the destructive one. Same pattern for CLEAR ALL SLOTS.
  • Roster cards re-typeset with a real hierarchy: name + status pill on top · 3 stat tiles · everything else collapsed behind STATS.
  • Seed becomes click-to-copy with a hover hint.
  • Money symbol removed (Pokédollars don't have a universal Unicode glyph).

A · STATE PILLS Always visible. EMPTY (border only) · SAVED (filled neutral) · ACTIVE (green) · TARGET (amber, only during overwrite mode) · CONFIRM (red, only mid-confirmation). The save state is glanceable without reading body copy.
B · ASYMMETRY OK The ACTIVE row hides the MAKE ACTIVE button (no-op anyway). Today this leaves a small visual gap; here we make the row narrower instead — DOWNLOAD and DELETE stretch.
C · EMPTY SLOT IS A CTA The empty slot's body copy is the call-to-action — "drop a save here" — instead of just the word "Empty" on a dimmed row.
D · CLEAR ALL SLOTS Dashed-border, bottom of column, separated by a divider. Reads as "yes this is destructive." First click triggers the same inline-confirm pattern (Screen 4).

User attempted to save from the emulator; all 5 slots were full. We surface a sticky banner explaining the situation and offering CANCEL. Targetable slots get the amber TARGET pill and an amber border. Clicking a slot writes there. The user can always back out.

A · BANNER IS THE ANNOUNCEMENT Replaces today's spring-loaded overlay. The user always knows they're in overwrite mode because the banner is right there.
B · TARGET PILL + BORDER Both border color (amber) and pill (amber) signal targetability. Color-blind safe because the pill text "TARGET" is also explicit.
C · ACTION BUTTONS HIDE While in TARGET mode, per-slot DOWNLOAD/MAKE ACTIVE/DELETE hide. The whole slot is now a click target. Removes the ambiguity of "what does clicking the slot do?"
D · ESCAPE HATCH CANCEL is on the banner, never more than one click away. ESC key does the same (focus-trap utility we add in Phase 1 of the audit).

Inline DELETE confirmation

User clicked DELETE on slot 3. Instead of opening a modal, the slot's action row swaps into an inline "Are you sure?" state. The destructive button is now red and labelled DELETE FOREVER. CANCEL returns to the previous state. No data lost on misclicks.

A · INLINE > MODAL The user is already looking at this slot. A modal would obscure the row and force a context switch. Inline confirmation reads as "I changed my mind one button to undo."
B · BUTTON LABELS DELETE FOREVER is more honest than CONFIRM DELETE. Tells the user what actually happens.
C · BODY COPY = STAKES "04 hours of progress" makes the loss concrete. If the slot is small, copy adapts to "30 minutes of progress" — keeps the cost legible.
D · KEYBOARD ENTER targets the cancel (safe-default), with TAB → ENTER for delete. ESC also cancels. Same pattern as MARK DEAD modal — consistency wins.

A · YOU MARKER 4px amber border + small "YOU" pill on the name. Survives Turbo Stream re-renders because it's CSS-only — no JS overlay needed.
B · 3-TILE STAT STRIP The three numbers everyone glances at first: BADGES (gym progress), DEX (caught), PLAY (hours). Readable from across the room. Other fields move into <details>.

D · HALL OF FAME Inline pill next to the trainer name. Small enough that it doesn't dominate but visible enough to celebrate.
